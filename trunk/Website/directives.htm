Directive Label Syntax Section LC Description
.START   .start,<program_name>,<initial_LC> .START No Indicates the start of the source code, the name of the program and the initial value of the Location Counter (LC).
.END   .end,<program_name> .END No Indicates the end of the program. The name of the program must be the same as the one in the start directive.
.DATA   n/a .DATA No Indicates the start of a section of code that defines data values. There can be several .DATA sections contained in the source code.
.TEXT   n/a .TEXT No Indicates the start of a section where actual "code" exists. There can be several .TEXT sections contained in the source code.
INT.DATA Optional  INT.DATA <Valid_Integer> .DATA Yes An integer value between -231 and (+231-1), and consumes a word of memory
STR.DATA Optional STR.DATA  <Valid_Character_String> .DATA Yes A string of characters enclosedin single quotes. The string can be a maximum of ****** characters long, and for memory purposes, the string is divided into groups of 4, with each group assigned a word of memory.
HEX.DATA Optional HEX.DATA  <Valid_Hex> .DATA Yes Hex string enclosed in single quotes, with a maximum of 8 hex digits. The acceptable range is 80000000 to 7FFFFFFF.
BIN.DATA Optional BIN.DATA  <Valid_Binary> .DATA Yes Binary string enclosed in single quotes with a maximum of 32 binary digits.
ADR.DATA Optional ADR.DATA <Label> .DATA Yes An internal or external label, referencing a valid memory address (maximum of 65,536).
ADR.EXP Optional ADR.EXP <Expression> .DATA Yes An expression that works out to have the value of an acceptable memory address. Expressions may contain constants, previously equated strings representative of numbers, local or external label references. Addition and subtraction are the only valid operators, and only 1 level of nesting is allowed. Only 3 labels/external references can be included.
ENT   ENT <Label>,<Label>,<Label>,<Label> .TEXT No Internal reference labels that must be defined in a .TEXT section, somewhere in the source code. The maximum is 4 labels.
EXT   EXT <Label>,<Label>,<Label>,<Label> .TEXT No External reference labels that may be used in the source code in any place where labels are valid operands. The maximum is 4 labels.
NOP Optional ol nop .TEXT Yes A NOP directive is the equivalent of the instruction "sll $0,$0,$0" and its purpose is to take up space/timing, but not affect the actual program.
EXEC.START   EXEC.START <Label> .TEXT No Allows the user to specify the exact memory address at which to start execution of the program. This allows for .DATA and .TEXT sections to be organises according to the programmer's wishes.
MEM.SKIP Optional MEM.SKIP <NNNN> .DATA Yes Reserves a specific amount of memory (quantity is in words) that is then skipped over by the LC. The reserved memory locations can then be used in the source code at the programmers's whim.
EQU Required  <Label> EQU <String> .TEXT No Equates a label with a valid string of data. The maximum is 32 characters.
EQU.EXP Required <Label> EQU.EXP <Expression> .TEXT Yes Equates a label with an expression that must evaluate to a valid data entry.
RESET.LC Required RESET.LC .TEXT Yes Resets the LC to a new, specified value. The new value must be greater than the last LC (i.e. the current one at this directive location)
DEBUG   DEBUG <N> .TEXT No Turns the debug flag on or off. Valid operands are 0 -- off and 1 -- on.
