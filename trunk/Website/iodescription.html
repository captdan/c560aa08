Directives:


Every input file must begin with a .start directive and conclude with a .end directive, no exceptions.

Other directives are used throughout to denote data or text areas and such. For examples on directives see the directives page.


SP1/2 Output:


Description of Object File: Holds the program name, its length, the date the program was run, the version of the program being run, the address where execution started, number of linking records including their addresses in hex, types and entry names. All program lines of code including their hex address, debugging code, number of adjustments, type of action (R,E or A), label references when present, and finally the end record which includes the number of header, linking, text and ending records.

Additionally each record is recorder on a separate line and the various fields of each record are separated by “|”. The object file will also be the input for SP3.


Assembly Listing: The assembly listing records the value of the location counter in HEX for each line of code, the object code for each line of code, the type of addressing (absolute, relative or external) the source code as it was read from the program file and any accompanying error for each source code recorded below the line of code where the error occurred.
