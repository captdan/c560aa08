Testing & Validation Philosophy

               

Our test philosophy consists of three steps:

 

1) Test correct code

                - We will test our code with syntactically correct code to ensure it will generate the correct output consistently.

2) Test incorrect code

                - Ensure our code will be capable of catching any syntax errors by the user without creating incorrect results.

3) Test all code

                - Instruction and directives will be tested, not only individually, but also with various combinations of instructions and directives.

               

Testing the User Code:

                We will use exhaustive testing based on test programs provided and our own test programs based on the specifications provided for SP1, SP2 as well as our own programs, we build specific programs to test individual instructions and directives, as well as some exhaustive test with every instruction.  We hand assemble the test code used and compare it to the output of our parser and the expected symbol table, and check for every syntax error we can account for in the time allowed.

Checking for errors:

                We look for errors mainly in our output files, which include the intermediate file, symbol table, object file, and report any errors found to the user in the assembly listing.
